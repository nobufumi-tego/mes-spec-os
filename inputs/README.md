# inputs/

利用者の **機密データ** を置く場所です。

## ここに置くもの（例）

- 自社の業務リスト（既存の Excel / PDF）
- 既存システム設計書・構成図
- 取引先・製品コード・ライン番号など、外に出せない情報
- 過去の MES / MOM 検討資料

## ここに置かないもの

- ENAA 公開資料 → `reference/ENAA/` に置く
- 生成された仕様書 → `outputs/` に保存される

## Git の取り扱い

`.gitignore` で本フォルダ配下のファイルは git の追跡対象から除外されています（本 README のみ追跡）。
誤って commit しないよう、`/保存` 時にも警告が出る設計です。

## AI へ渡すときの注意（**重要・必読**）

- ファイルを Claude に渡す前に、内容を必ず確認してください
- 個人名・取引先名・取引条件が含まれる場合は、伏せ字化を Claude が提案します
- 完全な原文を渡すかどうかは利用者の最終判断です（Claude は強要しません）

### ⚠️ Claude Code のデータフローの基本

- ローカルに置いた**だけ**では情報は外部に出ません
- ただし **Claude にそのファイルを処理させた瞬間、内容が Anthropic（米国）の API に送信されます**
- 「処理させる」とは：
  - Claude に「このファイルを読んで」と頼む
  - Claude が自発的に該当ファイルを参照する（CLAUDE.md などの指示で）
  - スラッシュコマンド（例 `/MES仕様書`）が内部で読み込みを行う

機密度の高いファイルは、まず伏せ字化したコピーを inputs/ に置くか、原本は Claude に処理させずに利用者自身で要点だけ伝える運用を推奨します。

詳細は [`docs/セキュリティ.md`](../docs/セキュリティ.md) を必ず確認してください。
