# 困ったとき

## まずこれを試してください

### エラーが出たら、Claude に貼り付けて聞く

```
[エラーメッセージをそのままコピー]
これ何？どうしたら直る？
```

これで多くの問題は解決します。

## それでもダメな場合

### Claude が起動しない

- Claude Pro 以上のサインインが完了しているか
- プランの quota 切れではないか（[Anthropic 管理画面](https://claude.ai) で確認）
- Node.js がインストールされているか（`node --version` で確認）

### ENAA 資料が認識されない

- `reference/ENAA/` 直下に 5 ファイルが揃っているか確認：

  ```powershell
  Get-ChildItem reference\ENAA\
  ```

- ファイル名が「ダウンロード.pdf」「Download.pdf」等になっていたら、Claude に **「ENAA 資料を確認して」** と頼めば自動識別＆リネームを提案してくれます

### ENAA からダウンロードできない（403 エラー等）

- ブラウザで ENAA 公式ページ https://www.enaa.or.jp/research/smart/mes を開いて、そこからナビゲートしてください
- リンクは **右クリック →「名前を付けて保存」**（直接クリックすると PDF/Excel ビューアで開いてしまい、ファイル名が「ダウンロード.pdf」になります）
- `?fname=...` 形式の URL（[reference/ENAA/README.md](../reference/ENAA/README.md) に記載）からは外部からも直接ダウンロードできます

### 文章を途中で改行したい / 入力途中で Enter を押してしまった

- 複数行の入力：**Shift + Enter** で改行
- 誤送信：そのまま続きを書けば Claude が前後をつなげて読み取ってくれます。「すみません、さっきのは入力途中でした、正しくは…」と書き直しても OK

### 文字化けする

PowerShell のコードページが Shift-JIS のままの可能性。

```powershell
chcp 65001
```

を実行してから再試行してください。

### Python 関連のエラー（uv / openpyxl / pypdf）

抽出レシピや cache.py で Python エラーが出る場合：

```powershell
uv --version       # uv が入っているか確認
```

uv が入っていなければ：

```powershell
winget install astral-sh.uv
```

文字化け系のエラーは `PYTHONIOENCODING=utf-8` で回避できます（Bash の場合は環境変数として、PowerShell の場合は `$env:PYTHONIOENCODING="utf-8"`）。

### 出力ファイルが outputs/ に保存されない

- Claude に「保存先のパスを教えて」と聞く
- 手動で `outputs\MES仕様書\` フォルダを作ってから再試行：

  ```powershell
  New-Item -ItemType Directory -Force -Path outputs\MES仕様書
  ```

### キャッシュがおかしい（古いデータが返る）

ENAA Excel を更新したら自動で再構築されますが、念のため強制再構築する場合：

```powershell
Remove-Item -Recurse -Force outputs\.cache
```

次回実行時に `cache.py` が新規構築します。

### inputs/ のファイルが間違って commit されそう

`.gitignore` で除外済みなので通常は大丈夫ですが、`/保存` 前に Claude が機密疑いの確認を入れてくれます。

万一 commit してしまった場合：

```powershell
git reset --soft HEAD~1   # 直前のコミットを取り消し
git restore --staged inputs/
```

## 機密情報の取り扱いで困ったとき

[docs/セキュリティ.md](セキュリティ.md) の **「7. リスクシナリオと対応」** を参照してください。

特に以下の状況：

- 機密が含まれた状態で Claude に渡してしまった
- outputs/ を public リポに push してしまった
- PC 紛失・盗難

これらの状況別の対応手順が書いてあります。

## サポートについて

本プロジェクトは個人運営の OSS で、**個別サポートは行っておりません**。詳細は [SUPPORT.md](../SUPPORT.md) を参照してください。

業務上の MES 導入支援が必要な場合は、有償の MES コンサルタント・SIer をご検討ください。

## 最後の手段

すべてやり直したい場合、フォルダごと削除して再 clone：

```powershell
cd $HOME\Documents
Remove-Item -Recurse -Force mes-spec-os
git clone https://github.com/nobufumi-tego/mes-spec-os.git
cd mes-spec-os
```

ただし `outputs/` は失われるので、必要なら先にバックアップを取ってください。
