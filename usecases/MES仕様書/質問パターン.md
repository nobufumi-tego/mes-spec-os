# MES 仕様書 質問パターン集

ENAA ガイド資料（G25-029-1 解説資料 / G25-029-3 クイックスタートガイド / G25-029-5 How to Use）から **抽象化された方法論・概念** を抽出し、雛形の対話品質を ENAA の意図と整合させるための質問テンプレート集。

> **設計思想**：ENAA の verbatim コンテンツ（用語定義文・全 Division 名列挙等）を取り込むのではなく、**概念フレーム** だけを要約・出典明記して活用する。これにより：
>
> - ✅ 著作権上の引用法理の範囲内に収まる
> - ✅ ENAA の意図する利用方向に対話を誘導できる
> - ✅ 出力が ENAA の体系と自然に整合する
>
> 本ファイル中の概念は ENAA 各資料からの **要約** であり、原文・図表は本リポに含まれない。詳細は利用者が ENAA 公式（https://www.enaa.or.jp/research/smart/mes ）から各資料を取得して参照する設計。

## ENAA 概念リファレンス

雛形の対話で活用する 6 つの ENAA 概念：

### 概念 A: フェーズ区分（出典: G25-029-3 P10）

ENAA は MES 検討を以下のフェーズに分解：

```
[企画・要件定義]                                  [開発] [運用] [保守]
 システム化の方向性 → システム化計画 → 要件定義 → ...
                       ★ 本雛形の対象 ★
```

**「システム化の方向性」**（MES を入れるべきか）と **「システム化計画」**（具体的な範囲・要件）の区別が肝。本雛形は後者を対象とする。

### 概念 B: MES対象記号 4 値（出典: G25-029-1 P9-10 表 3-2）

ENAA Excel の MES/MOM 対象列で使われる記号：

| 記号 | 意味 | 雛形での解釈 |
|:---:|---|---|
| ○ | MES/MOM で対応可能な業務プロセス | **必須** で MES に含める |
| △ | 対象の場合あり（一部業務で活用するケース等） | **任意** で MES に含めるかは利用者判断 |
| × | MES では対象としない業務プロセス | **不要**（他システム or 業務で対応） |
| ※ | 別の Division 内で記載がある内容（重複参照） | **別 Division 参照**（重複登録しない） |

雛形 Stage 1 の Division 必要性判定はこの 4 値と紐付ける。

### 概念 C: 代替可能システム 13 種（出典: G25-029-1 P10-11 表 3-3）

MES でやらない場合の代替先として ENAA が定義する 13 種類：

```
SCP / ERP / スケジューラー / WMS / IoT・SCADA・DCS / CAPA /
QMS / LIMS / PLM / BI / CMMS / HRIS / SRM
```

各システム名の正式名・補足は ENAA G25-029-1 P10-11 を参照（本雛形では verbatim 引用しない）。Stage 2 の代替システム判定で選択肢として活用。

### 概念 D: MES/MOM の 3 効果軸（出典: G25-029-3 P14）

ENAA が示す MES/MOM 導入効果の 3 つの軸：

1. **生産性・品質の向上**（定量化しやすい）— 既存能力の効率化
2. **新しい能力の獲得**（定量化しやすい）— 今までできなかった業務を可能に
3. **判断の質の向上**（定量化しにくいが重要）— カーナビ的、データに基づく意思決定

**カーナビメタファー**（ENAA 提案）：MES は速度や燃費を直接上げるのではなく、現在地・残距離・渋滞情報を提供することで「迂回判断」「先方への連絡」「再計画」など意思決定の質を変える。

### 概念 E: 利用者カスタム 6 項目（出典: G25-029-3 P13）

ENAA が ENAA Excel の P 列以降に追加することを推奨する 6 項目：

```
担当部門名 / 該当業務あり〇 なし× / 現業務媒体（システム名・Excel・手書き等）/
現状業務・処理内容 / 現状業務における課題 / 改善・実現したいこと（To-Be）
```

雛形 Stage 1 の Business Category 深掘り質問の **標準セット** として活用。

### 概念 F: ENAA 認識の未対応領域（出典: G25-029-1 P19）

ENAA が「現時点で議論不十分」と認める領域 6 つ：

```
非機能要件 / 外注管理 / KPI などの管理指標 / カンバンの運用 /
工場間を跨いだプロセス / リサイクル・SDG
```

雛形 Stage 4 で **「本仕様書の射程外」として明示** する範囲（ベンダーへの誤解を防ぐ）。

## Stage 0：前提確認・案件メタ聴取の質問パターン

### Q0-1: フェーズ確認（概念 A 活用）

```
ご検討中の MES/MOM 導入は、現在どのフェーズにありますか？

  a) システム化の方向性を検討中（MES 入れるか自体まだ決まっていない）
     → 本雛形の射程外です。ENAA 出版予定の書籍等の活用をご検討ください
  b) システム化計画を立てている（MES 入れることは決まり、範囲・要件を整理中）
     → 本雛形の主たる対象です
  c) 要件定義以降のフェーズ（既に RFP 発行・ベンダー選定済み）
     → 本雛形の出力は参考情報レベルです
```

→ b の場合のみ Stage 1 へ進む。a/c の場合は雛形のスコープ外を伝えて中断オプション。

### Q0-2: 工場タイプ確認

```
ENAA 標準業務一覧は組立加工系（discrete）工場を主に想定しています。
貴社の対象工場のタイプは？

  a) discrete（組立加工系：自動車部品・電子機器等）→ 本雛形の主対象
  b) process（プロセス系：化学・石油等）→ 出力は ENAA 標準準拠ではなく参考レベル
  c) batch（バッチ系：医薬品・食品等）→ 同上
  d) 不明 / 混在
```

### Q0-3〜0-7: 案件メタ（規模・期間・予算・担当）

```
案件名・対象工場の所在・従業員数・ライン数・主要顧客・期間・予算感・担当部門
```

→ 1 問 1 答で順次聴取（業務改善OS の「成果先行」原則と整合）。

## Stage 1：調査星取表の質問パターン

### Q1-1: Division 必要性判定（概念 B 活用）

ENAA 12 大分類のそれぞれについて、以下を聞く：

```
Division「<名称>」について、貴社では MES/MOM 対象としてどう扱いますか？

  ○ 必須（MES でやる、対象として含める）
  △ 任意（MES でもいいが、他システム代替も検討の余地あり）
  × 不要（MES では対象としない、他システム or 業務で対応）
  ※ 別 Division 参照（既に他 Division で扱う）
  ? 未定（情報不足、後で判断）
```

→ ENAA の 4 値（○/△/×/※）に「未定」を追加した 5 値。具体名選択肢にすることで、利用者が ENAA の体系を自然に体得できる。

### Q1-2: Business Category 深掘り（概念 E 活用）

「○ 必須」「△ 任意」と判定された Division に対して、その配下の Business Category 単位で **ENAA 推奨カスタム 6 項目** を聞く：

```
Business Category「<名称>」について、ENAA 推奨カスタム 6 項目で整理します：

  1. 担当部門は？
  2. 該当業務はある？（あり○ / なし×）
  3. 現業務媒体は？（既存システム名 / Excel / 手書き / その他）
  4. 現状業務はどう行っている？（簡潔に）
  5. 現状の課題は？（複数可）
  6. 改善・実現したいこと（To-Be）は？
```

→ ENAA Excel の P 列以降に書く想定の項目を、対話で聞いて outputs/ YAML に整理。

### Q1-3: 周辺 Division の一括判定

ENAA 4 値分類（○ + △ × 0.5）/ 総数 の重み比率で、Division は概ね 3 グループに分かれる：

| グループ | 重み比率 | Division |
|---|:---:|---|
| **中核 5**（80% 以上、ほぼ MES 対象） | 80-100% | 製造 / 品質管理 / 物流・在庫管理 / 設備保全 / 品質保証 |
| **中間 4**（30-80%、ケースバイケース） | 32-73% | 従業員 / 生産管理 / 生産技術 / HSE |
| **周辺 3**（30% 未満、MES では基本扱わない） | < 30% | 設計 / 購買・資材 / 工場経理 |

```
以下 3 Division は MES/MOM 対象比率が極めて低い領域です（既存 ERP 等で完結することが多い）。
一括で「× 不要」で進めてよろしいですか？それとも個別に確認しますか？

  - 設計（製品設計、自社で行う場合のみ）
  - 購買・資材（既存 ERP で完結する場合が多い）
  - 工場経理（既存 ERP で完結する場合が多い）
```

→ 周辺 3 Division は MES 対象比率が低いため一括処理。一方、**HSE / 生産技術** は ENAA Excel で △（対象の場合あり）が多く、業種・状況により判断が分かれるため必ず個別確認すること。

## Stage 2：要求機能整理の質問パターン

### Q2-1: 課題から機能への落とし込み

```
Stage 1 で挙がった以下の課題を、要求機能として整理します。
各課題について、To-Be 施策（解決策・実現したいこと）を教えてください。

  課題 1: <Stage 1 で記録された課題>
  → To-Be 施策は？
```

### Q2-2: 代替可能システム照合（概念 C 活用）

```
この機能を MES でやらない場合、ENAA が示す 13 種類の代替可能システム
（SCP / ERP / スケジューラー / WMS / IoT・SCADA・DCS / CAPA / QMS /
LIMS / PLM / BI / CMMS / HRIS / SRM）のいずれかで対応できそうですか？

  - 既存システムで代替可能 → どの種別？
  - 新規導入が必要 → MES と一体か別契約か？
  - 業務（人手）で代替 → 課題が継続するリスクあり？
  
詳細な定義は ENAA G25-029-1 P10-11 表 3-3 をご参照ください。
```

### Q2-3: 優先順位付け

```
要求機能を以下の観点で評価してください：

  - 効果の大きさ（高/中/低）
  - 早さ（短期 1 年以内 / 中期 2-3 年 / 長期 4 年以上）
  - 難易度（低 / 中 / 高）
  
これらを総合して、優先度（高/中/低）と実現時期を仮置きします。
```

## Stage 3：期待効果と ROI の質問パターン

### Q3-1: 3 効果軸での整理（概念 D 活用）

```
要求機能リストを、ENAA が示す 3 効果軸で整理します。
各機能はどの軸に該当しますか？（複数可）

  軸 1: 生産性・品質の向上（効率化、定型業務削減 → 定量化しやすい）
  軸 2: 新しい能力の獲得（今できないことを可能に → 定量化しやすい）
  軸 3: 判断の質の向上（カーナビ的、データに基づく意思決定 → 定量化困難）
```

### Q3-2: カーナビメタファー誘導（概念 D 活用）

「軸 3: 判断の質の向上」の効果は定量化困難なので、ENAA のカーナビメタファーで具体例を引き出す：

```
「判断の質の向上」効果は、カーナビが速度を直接上げるのではなく、
現在地・残距離・渋滞情報を提供して「迂回」「再計画」「事前連絡」など
意思決定を変えることに似ています。

貴社の場合、この機能で「今まで知らなかった」「数時間後にしか分からなかった」
情報がリアルタイムで分かることで、現場や管理者の判断がどう変わりそうですか？
```

### Q3-3: ROI 概算の聴取

```
投資見積（初期 + 保守）と年間効果（3 軸合計）を概算で聞きます。
投資見積が出ていない場合は、目標 ROI 年数からの逆算で投資上限を提示します。
```

## Stage 4：ロードマップ・RFP 一次稿生成の質問パターン

### Q4-1: Phase 設計

```
Stage 2 の要求機能を、優先度・実現時期で 2〜3 Phase に分けます。
推奨構造：

  Phase 1（短期）: 効果が早く出る機能（製造実績可視化等）
  Phase 2（中期）: Phase 1 のデータを前提にする機能（品質トレース等）
  Phase 3（長期）: 設備改造を伴う機能（IoT 化等）
```

### Q4-2: 非機能要件聴取

```
パフォーマンス・可用性・セキュリティ・既存基幹系連携の各観点で要件を整理します。
```

### Q4-3: ENAA 未対応領域の明示（概念 F 活用）

```
本仕様書の **射程外** として明示する領域：

  - 非機能要件の詳細水準（ENAA でも継続検討）
  - 外注管理（ENAA でも継続検討）
  - KPI などの管理指標（同上）
  - カンバンの運用（同上）
  - 工場間を跨いだプロセス（同上）
  - リサイクル・SDG（同上）

これらが必要な場合、別途要件定義フェーズで深堀りすることを RFP 一次稿に
明記します。よろしいですか？
```

→ ENAA が認識する未対応領域を **明示的に射程外** とすることで、ベンダーとの認識齟齬を予防。

### Q4-4: RFP 想定

```
発行時期 / 対象パートナー候補 / 選定基準
```

## 既存 outputs サンプル（マルヨシ精工）との対応

`outputs/MES仕様書/20260502_マルヨシ精工_MES導入仕様書.{yaml,md}` のサンプルでは、簡易 3 値（必須/任意/不要）を使用している。これは ENAA 4 値との以下のマッピング：

| サンプルの値 | ENAA 4 値 |
|---|:---:|
| 必須 | ○ |
| 任意 | △ |
| 不要 | × |
| (使用なし) | ※（別 Division 参照） |

新規案件では本ファイルの 4-5 値（○/△/×/※/未定）使用が推奨。
