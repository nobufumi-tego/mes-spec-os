# ENAA 資料の取得と配置

ENAA「MES/MOM 導入のための標準業務一覧 G25-029 シリーズ」（2025 年 9 月正式版）を、本フォルダ（`reference/ENAA/`）に配置してください。
本スターターはこれらの資料を **前提**として動作します。

## なぜ利用者が手動でダウンロードするのか

- ENAA 資料は再配布が許可されていない可能性がある（自社内・RFP 二次利用は明示許可、リポジトリ同梱・再配布は明示されていない）
- 著作権表示の責任を本スターター側で持ちきれない
- 公式サイトから取得することで利用者が利用条件を直接確認できる

## 取得元（ENAA 公式サイト）

ブラウザで以下のページを開いてください。各資料へのリンクが掲載されています：

  https://www.enaa.or.jp/research/smart/mes

## ダウンロードのコツ（重要）

**リンクを直接クリックすると、ブラウザの内蔵 PDF / Excel ビューアで開いてしまい、「ダウンロード.pdf」「Download.pdf」のような既定名で保存される事故が起きやすいです。**
これだと 4 つの PDF がどれも同じような名前になって、どれが G25-029-1〜5 なのか後から判別できなくなります。

必ず以下の方法でリンク先のファイル名のまま保存してください：

| ブラウザ | 推奨操作 |
|---|---|
| Microsoft Edge | リンクを **右クリック → 「名前を付けてリンクを保存」** |
| Google Chrome | リンクを **右クリック → 「名前を付けてリンク先を保存」** |
| Firefox | リンクを **右クリック → 「名前を付けてリンク先を保存」** |
| Safari (Mac) | リンクを **option + クリック**（または右クリック →「リンク先のファイルをダウンロード」） |

これで `mes_g25-029-1.pdf` 等のオリジナル名で保存できます。

### もし違う名前で保存してしまったら

`reference/ENAA/` に「ダウンロード.pdf」「Download.pdf」「download (1).pdf」等の名前で置いてしまっても、捨てて再ダウンロードする必要はありません。
Claude を起動して **「ENAA 資料を確認して」** と頼むと、PDF の中身を読んで自動で識別 → 正しい名前へのリネームを提案してくれます（仕組み：`core/rules.md` 「ENAA 資料の状態確認とフォールバック」）。

メニューから `2) ENAA 資料の確認` を選んでも同じ動作になります。

## 配置先

ダウンロードした 5 ファイルを **本フォルダ（`reference/ENAA/`）の直下** に置いてください。
ファイル名は ENAA のままで変更しないでください（スターター側はこの名前で参照します）。

配置後の状態（理想形）：

```
reference/ENAA/
├── README.md             ← この説明ファイル（既存・gitに追跡される）
├── mes_g25-029-1.pdf     ← 利用者が配置
├── mes_g25-029-2.xlsx    ← 利用者が配置（最重要）
├── mes_g25-029-3.pdf     ← 利用者が配置
├── mes_g25-029-4.pdf     ← 利用者が配置
└── mes_g25-029-5.pdf     ← 利用者が配置
```

`.gitignore` で資料本体は git の追跡対象から除外されています（本 README のみ追跡）。

## 著作権・利用条件（重要）

ENAA 公式サイトに記載されている条件：

- **自社内利用・RFP への二次利用 可**
- **著作権表示必須**（一般財団法人エンジニアリング協会の名称を明示）
- 再配布の可否は明示されていない（→ 本リポジトリに同梱しない方針の根拠）

このスターターを使って生成される仕様書には、ENAA 出典が末尾に**自動で付記**されます（`CLAUDE.md` / `core/rules.md` で強制）。
削除しないでください。

## バージョン管理

ENAA は継続的に改善・更新を予定しています。長期間使用する場合：

- 定期的（半年に 1 回程度）に公式サイトを確認して更新版に差し替えてください
- ダウンロード時のバージョン（例：「2025 年 9 月正式版」）を `version.txt` 等にメモしておくと、後で出力のトレーサビリティが保てます

## 配置できているか確認するには

PowerShell で本フォルダに移動して：

```powershell
Get-ChildItem reference\ENAA\ | Select-Object Name, Length
```

5 ファイル全部が表示されれば OK。
