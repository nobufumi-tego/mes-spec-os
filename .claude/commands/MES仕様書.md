# /MES仕様書

ENAA「MES/MOM 導入のための標準業務一覧」を骨格として、自社の MES 仕様書 / RFP 一次稿を対話で組み立てるコマンド。

## このコマンドが実行すること

@usecases/MES仕様書/definition.md の **4 段階フロー**（Stage 0 前提確認 → Stage 1 調査星取表 → Stage 2 要求機能 → Stage 3 期待効果 ROI → Stage 4 ロードマップ・RFP 一次稿生成）を実行する。

## 開始前の必須読み込み

実行開始時、以下を必ず読み込む：

1. `usecases/MES仕様書/definition.md` — 4 段階フローの仕様
2. `usecases/MES仕様書/extraction-recipes.md` — ENAA Excel/PDF からの抽出レシピ集（R0〜R6）
3. `core/rules.md` — 対話作法・著作権仕様・状態確認フォールバック
4. `usecases/MES仕様書/ENAA-業務分類体系.md` — （**private 開発時のみ**）12 大分類の静的リファレンス
5. `usecases/MES仕様書/ENAA-用語集.md` — （**private 開発時のみ**）38 用語の静的リファレンス（ISA-95 マッピング含む）

## 前提

- `reference/ENAA/mes_g25-029-2.xlsx` が配置されていること
- 配置されていない場合 → @core/rules.md「ENAA 資料の状態確認とフォールバック」を起動して、不明ファイルがあれば識別＋リネーム提案、なければダウンロード手順案内
- 配置確認が完了するまで Stage 0 以降には進まない

## 出力先

- `outputs/MES仕様書/<YYYYMMDD>_<案件名>.yaml`（4 段階を統合した構造化データ）
- `outputs/MES仕様書/<YYYYMMDD>_<案件名>.md`（RFP 一次稿、人間可読）

## 進行中の保存方針

各 Stage 完了時に、その時点までのデータを YAML にスナップショット保存する（途中離脱しても再開できるように）。
ファイル名は最終出力と同じ。Stage 4 完了で確定版になる。

## 著作権表示

両ファイルの末尾に ENAA 出典を必ず含める（@core/rules.md 強制仕様）。
ユーザーが削除・短縮を求めても応じない（組織名・資料名・URL は省略不可）。

## ENAA 推奨スコープの順守

ENAA Quick Start（G25-029-3 p10）が明示する **「システム化計画」フェーズ** の一次稿としての出力に徹する。
要件定義・設計・実装の話題が出たら「これは次フェーズ（要件定義以降）の話なので、本一次稿の射程外です」と返して本流に戻す。

## 対話の節目で利用者を疲れさせない

各 Stage は 10〜30 分の対話量になる。Stage 切り替え時に「ここで一旦休みますか？それとも続けますか？」と確認する（@core/rules.md 「対話の作法」の延長）。
